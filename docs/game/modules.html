However, for this game we will use cardinal directions to represent the factions. with the "north" and "south".

Within the game files, we will use player information for each "faction", but as this can be dynamic, we will use the cardinal directions to represent the factions within the code.

    • Defined in lib/utils/on-move/get-move-locations.ts:19

    Returns the available board-locations that the plane can move to.

    1. A plane must move the entire dice roll.
    2. A plane cannot move over the same location 2 times, except stopping at the starting location.
    3. A plane cannot move through another plane.

    Parameters

    • __namedParameters: { dice: DiceSides; inFlightPlanes: Pick<Plane, "x" | "y">[]; plane: Pick<Plane, "x" | "y"> }

      • inFlightPlanes: Pick<Plane, "x" | "y">[]

        The list of planes currently in flight

      • plane: Pick<Plane, "x" | "y">

        The plane we are returning moves for.

    Returns { boardLocationStrs: BoardLocationStr[]; boardLocations: BoardLocation[] }

      The location strings that the plane can move, this is provided more as a helper for situations where you need to perform lookups against what was returned, as the board-lookup-str
